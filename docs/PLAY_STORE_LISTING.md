## Graphic Assets

### App Icon Requirements

**Specifications:**
- **Size:** 512√ó512 pixels (high-resolution PNG)
- **File size:** < 1 MB
- **Format:** 32-bit PNG with alpha transparency
- **Color space:** sRGB
- **No rounded corners:** Google Play automatically applies adaptive icon masking

**Design Concept for DebugMaster Icon:**

**Primary Concept: "Bug Under Magnifying Glass"**
- **Background:** Vibrant gradient from `#4CAF50` (success green) to `#2196F3` (tech blue)
- **Main element:** Large, stylized bug silhouette (üêû) in center, clean geometric shape
- **Accent:** Magnifying glass partially overlapping bug, suggesting "finding/fixing"
- **Style:** Flat design, minimal, highly readable even at small sizes
- **Typography:** Bold "D" or "DM" monogram in corner (optional, if icon feels empty)

**Alternative Concept: "Code with Fix"**
- **Background:** Dark gradient (`#1E1E1E` to `#2C2C2C`) representing code editor
- **Main element:** 3-4 lines of simplified "code" (colored horizontal bars)
- **Accent:** One line crossed out with a bright green checkmark replacing it
- **Style:** Modern, developer-focused, techy aesthetic

**Why it works:**
- Instantly recognizable as debugging/bug-related
- Simple enough to be readable at 48√ó48px (launcher icon size)
- Colors align with Material Design and success/tech vibes
- Unique in the education/programming category

### Screenshot Guidelines

**Specifications:**
- **Format:** JPG or PNG
- **Dimensions:** Minimum 320px on shortest side, maximum 3840px on longest side
- **Aspect ratio:** 16:9 or 9:16 recommended for modern phones
- **Quantity:** 2-8 screenshots required (8 recommended for best presentation)

**Best Practices:**

1. **Use device frames:** Show screenshots in phone frames (Pixel, Samsung) for professional look
   - Tools: Figma, Photoshop, or online tools like MockuPhone

2. **Add text overlays:** Short, compelling captions on each screenshot
   - Font: Bold, sans-serif, high contrast
   - Position: Top 15% or bottom 15% of image
   - Background: Semi-transparent bar for readability

3. **Show the flow:** Order screenshots to tell a story
   - Start with onboarding/value prop
   - Show core features
   - End with results/achievements

4. **Readable code:** Ensure code text is large enough to read
   - Zoom in on important UI elements
   - Use high-DPI screenshots (2x or 3x)

5. **Show state diversity:**
   - Mix of empty states and populated screens
   - Show different difficulty levels
   - Display various achievements (locked and unlocked)

6. **Localize if needed:** Create separate screenshots for each supported language

**Tools to create screenshots:**
- Android Studio Screenshot tool (Device File Explorer)
- ADB command: `adb shell screencap /sdcard/screenshot.png`
- Figma or Adobe XD for mockups with text overlays
- Fastlane Frameit for automated device framing
